# シビックテック井戸端キャスト

# 自己紹介

Code for SOKAの牧内です。4月に立ち上がったばっかりで、先日この番組に出演していた河津さんが主導してくれました。

草加市を主な活動としている団体で、「ちょっといいミライをプロトタイプする」をモットーに地域課題を解決していこう！と考えています。

現在はアプリを中心とした受託業務のマネージャ業に従事していますが、今でもアプリを作ったりしている関係で、設計チームとエンジニアチームの橋渡し的なことをやることが多いです。仕事ではプログラムに手を出しにくいので、本当に切羽詰まった時しか手を出してません。そのかわりシビックテック活動の中でプロトタイプを作って楽しんでます。

# シビックテックとの関わり

最初に関わったのは、東日本大震災の後です。

その前から、Androidの普及促進をするグループがありまして、その会に入ってました。目的は、会社の業務ではできないスマホアプリを作ってみたかったからです。私がエンジニアだからか、想像力がないからか、作りたいものがないんです。だから、誰かが言った作りたいものを一緒に作りたいなと思って。

そうしたら、東日本大震災が起きまして、その直前からAndroidアプリで、道路の凸凹を計るアプリを作っている人が、一緒に作りませんかと。それはぜひ、となって一緒にアプリ作成をし始めました。これが最初ですね。

このアプリは、加速度センサーとGPSでどこでたくさん揺れたか、その揺れはだいたいどのくらいの段差で起きたのかを計測するもので、すごく面白いものだったのと、震災に関連したものだったので、A3 Togetherで大賞をいただきました。テーマが復興だったので。

このアプリを放っておいたらもったいないと思ったので、ベンチャーを作ってこのアプリを広げようってことになったのですが、そこからは結構今のシビックテック界隈の人たちと関係ができました。

とは言ってもビジネスに精を出していたので、活動は誘われればやる、といった程度でした。

ちょっと自分でもやりたいな、と思い始めたのがコロナになってからですかね。

自分で始められないかな、と考えている中、Code for Japanがソーシャルテクノロジーオフィサー創出プロジェクトをする、と耳にして参加しました。これはNPOの方針、指針を考慮してIT技術の導入に責任を持つ人を作り出し、NPOのテクノロジー利活用の推進をする、というプロジェクトでした。面白そうなので参加したところ、コロナの影響で対面の面談ができず、Zoomはハードルが高いし、どうしよう、というNPOとマッチングしました。Alexaを使って、スマートスピーカーで面談をするという仕組みを提供しました。Alexaスキルをいじって、伝言メモが残るようにしたのも楽しかったです。

そのうち地元の草加でやれないかなと考え始め、このほどCode for SOKAができて参加している、という状況です。

# 牧内さんの現在のお仕事や困りごと

アプリの受託開発のプロジェクトマネージャーをやってる。アプリを使ったマーケティングサービスを展開している会社なので、それなりの規模のプロジェクトが動いている。

それなりの規模の会社で、それなりの規模のプロジェクトなので、どうしても人数で見積もるので、チームビルドが重要になってくると考えているが、内部の組織間に距離があるように感じる。その距離をどう縮めるのか、というのも現在の課題としてある。

もう一点、現在はプロジェクトマネージメント部門にいるが、この部門に入ってくる人たちのバックボーンが様々で、エンターテイメント系とSIケイトで文化が違うが、この違いが認識の齟齬になっていくことがあるように感じている。自社のPM像をしっかり作り上げる必要があるんじゃないかな、と。

# 今までのキャリア的なお話

最初は、Windows95をきっかけに、パソコンに興味を持ったので、今で言うSESの会社に入った。ど素人だったので、3ヶ月くらいかけてUnixとC言語の研修を受けた。現場に出るときに3年と書かれていて、びっくりした。そんな感じなので、現場で通用するわけもなく、かといってサポートがあるわけでもなく、なかなか辛い状況だったところで、頭をけがして何針か縫う事態になったので、それを機会にやめた。次が、汎用機系の現場に人を出す会社がWindows系の開発をやり始めるということで、小さな会社にジョイン。ここは社内の社長の評判がすこぶる悪かったけど、やりがいはあったし、その後のキャリアを作る人たちとの出会いがあった。

その会社で上司と先輩だった人たちと会社を作ったのがその次で、システムの受託開発やSESをやってた。ここである会社のSEとして常駐してたが、情報システム部門なしの子会社を作ることになって、そこに参加した。会社ってこんな感じで作れるんだ、という特殊な経験をしたが、これもその後のキャリアに大きな影響を与えている。

2011年3月の東日本大震災の時に、スマホのセンサーを使って道路の凸凹を記録するというアプリを作る、プロボノというか、この時はボランティアというよりは、面白い技術でアプリを作る仲間みたいな感じで、開発チームができて、アプリを作ってた。この時、自分の持ってる技術が社会に役立つことを知った。このアプリは災害時に役立つが、それだけだといざという時使えない、ということで、このアプリを使ったビジネスでベンチャーを立ち上げた。ここからコードフォーとか、シビックテックとかプロボノとかに興味を持って参加するようになっていく。あと、海外の学会やシンポジウム、展示会などにたくさん参加した。

このベンチャーを含めたスマホアプリの経験を使って、チーム開発の高速化みたいなことをやってみたくて、知り合いの会社がマネージャーがいなくて困っているのもあり、受託開発の会社に参加した。失敗したプロジェクトを引き受けてくる社長だったので、いろいろ大変だったが、高速回転する開発チームはビルドできたんじゃないかな、と思う。

その経験をもっと大きな会社で試してみたくて、現在に至る。

# プロジェクトマネジメントで課題に感じること

自走するチーム、自ら考え行動するメンバーという状態が最高だと考えるが、それができるのは小規模なチーム。これを複数作って大規模案件ができないかというところが課題。

現在関わっている案件は、サイズが比較的大きく、関わる人数も多いが、その分意思決定が遅くなったり、会議が増えて手を動かす時間が減ったりする。できるだけ非同期に作業を行いたいができていない。

非同期ファーストで行きたい。
