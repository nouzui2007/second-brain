# G空間・DCS自動連携

## 処理フローの詳細

### データフロー概要

1. DCSメニュー生成バッチが、G空間センターからCKAN APIを使って直近7日間の更新データを取得
    - 対象：「登記所備付地図」または「大阪市航空写真」の文字列をタイトルに含むデータセット
2. 取得したデータセットごとに以下の処理を実行:
    - リソース格納S3からリソースファイルを取得
    - GeoJSONまたはGeoTiffファイルの場合のみ、以下の変換処理を実行:
        - 変換先S3に変換元ファイルを配置
        - 変換関数（鈴木さん実装）を呼び出し
        - 変換関数が変換済みファイルを変換先S3に配置
        - 変換関数が変換済みファイルのURLを返却
        - DCSメニュー生成バッチがメニューを生成

## 更新リスト作成→連携用データセット登録（手動）→メニュー生成バッチ

### 全体の流れ

```mermaid
sequenceDiagram
	participant updater as 更新リスト作成バッチ
	actor user as 手動
	participant ckan as CKAN
  participant generator as DCSメニュー生成バッチ
  
  updater ->> ckan: 変更データ取得[直近7日]
  ckan -->> updater: データセット群
  loop 取得したデータセット群
	  alt 法務省地図
		  updater ->> updater: 変換処理
		  updater ->> ckan: 連携用データセット更新[タイトル：法務省地図]
		else 大阪市航空写真
			updater ->> updater: 変換処理
		  updater ->> ckan: 連携用データセット更新[タイトル：大阪市航空写真]
		else 国土数値
		  updater ->> ckan: 連携用データセット更新[タイトル：国土数値]
		else
		  updater ->> updater: 更新リスト生成
	  end
	end
	updater ->> user: 更新リスト
  user ->> ckan: 連携用データセット登録
  generator ->> ckan: 定期巡回
  opt 更新あり
	  generator ->> ckan: データセット取得
	  generator ->> generator: DCSメニュー生成
  end

```

### 連携用データセット生成バッチ

```mermaid
sequenceDiagram
	participant sender as 連携用データセット生成バッチ
	participant geospatial as G空間センター
  participant RS3 as リソース格納S3
  participant SS3 as s3://ckan-storage-n
  participant CS3 as s3://gic-public-convert-data/ckan-strage-n
  participant geojson as GeoJSON変換関数（鈴木さん実装）
  participant geotiff as GeoTiff変換関数（鈴木さん実装）

	note over sender, geospatial: データセットの更新分を取得
	sender ->> geospatial: CKAN API（前日更新分）
	geospatial -->> sender: データセット群
	loop データセット群
		alt 国土数値、法務局備付地図、航空写真
			sender ->> sender: リソース作成
			opt 連携用データセットがない
				sender ->> geospatial: 連携用データセット登録
			end
			sender ->> geospatial: リソース登録
		else
			sender ->> sender: 更新リスト追加
		end
		sender ->> RS3: リソースファイル取得
		sender ->> SS3: 変換元として配置
		opt GeoJSON
			sender -) geojson: https://i9bwj63p0h.execute-api.ap-northeast-1.amazonaws.com/v1/jobs
			geojson --) sender: 受付
			geojson ->> SS3: ファイル取得
			geojson ->> geojson: 変換
			geojson ->> CS3: ファイル配置
		end
		opt GeoTiff
			sender -) geotiff: https://om5mmmdn75.execute-api.ap-northeast-1.amazonaws.com/convert
			geotiff --) sender: 受付
			geotiff ->> SS3: ファイル取得
			geotiff ->> geotiff: 変換
			geotiff ->> CS3: ファイル配置
		end
	end

```

### メニュー生成バッチ

```mermaid
sequenceDiagram
	participant sender as メニュー生成バッチ
	participant geospatial as G空間センター
	participant menu as メニューファイル
	participant config as Configファイル
  participant RS3 as メニュー格納S3

	loop 都道府県
		note over sender, geospatial: 連携用のデータセットの更新分を取得
		sender ->> geospatial: CKAN API[更新日=前日、カテゴリ=連携用グループ、エリア=処理中の都道府県）
		geospatial -->> sender: データセット群
		loop データセット群
			loop 市区町村
				alt メニューファイルなし
					sender ->> menu: メニュー作成
				else
					sender ->> menu: ファイルオープン
				end
				opt カテゴリが存在しない
					sender ->> menu: データセットタイトルでカテゴリ追加
				end
				alt Configファイルなし
					sender ->> config: Config作成
				else
					sender ->> config: ファイルオープン
				end
				loop リソース
					opt リソースの市区町村が一致
							sender ->> sender: リソース内のURLでファイル取得
							sender ->> sender: ファイルの内容確認
							alt GeoJSON
								sender ->> sender: 既存のGeoJSON用の処理
		　　					sender ->> config: リソースの内容で設定を追加
								else PMTiles
								sender ->> sender: 既存のPMTilesの処理
		　　					sender ->> config: リソースの内容で設定を追加
							else 3D点群
								sender ->> sender: 既存の3D点群の処理
		　　					sender ->> config: リソースの内容で設定を追加
							else
								note over sender: 何もしない
							end
					end
				end
				sender ->> menu: メニュー保存
			end
		end
		loop 作成したメニュー分
			sender ->> RS3: メニュー取得
			sender ->> sender: メニューマージ
		end
		sender ->> RS3: メニュー配置
	end	

```
