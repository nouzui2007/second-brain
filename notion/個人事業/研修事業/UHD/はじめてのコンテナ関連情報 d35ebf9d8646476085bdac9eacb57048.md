## Dockerコマンド

インストールされているバージョンを確認する。

```
docker –version
```

Dockerコマンドのヘルプを表示する。

```
# dockerコマンドに対するヘルプdocker –help# コンテナに対するコマンドのヘルプdocker –contianer –help# execのヘルプdocker exec –help# runのヘルプdocker run --help
```

### Hello World Docker

```
docker run hello-world
```

### Dockerのライフサイクルごとのコマンド

### 起動

コンテナを起動する。

```
# 最新版のイメージで起動docker run –p 8080:80 nginx# バージョンを指定したイメージから起動docker run --name oldnginx -d -p 8081:80 nginx:1.10.3-alpine# コンテナの内部で操作できるように起動（コンソールを譲る）docker run –it nginx bash# バックグランドで起動docker run –d nginx# バックグランドで起動して、停止後はコンテナを消すdocker run –d –-rm nginx
```

### 起動中

コンテナの状況を確認する。

```
# 起動しているコンテナの状態を確認docker ps# 全てのコンテナの状態を確認docker ps -a
```

稼働中のコンテナを操作する。

```
# コンテナに日付を表示docker exec nginx_container date# コンテナ内のnginxの設定ファイルを表示docker exec nginx_container cat  /etc/nginx/nginx.conf# コンテナに入ってbashを使うdocker exec –it nginx_container bash
```

コンテナの詳細情報を確認する。

```
# 詳細情報を確認docker inspect nginx_container# レイヤーを確認する$ docker history flask2
```

ログ出力を見る。

```
docker logs nginx_container# ログを出力し続ける（止めるときはCtrl + C）docker logs –f nginx_container# 時刻付きで出力docker logs –t nginx_container
```

ホストからコンテナにファイルをコピーする。

```
docker cp hello.py base:/
```

### 停止

稼働しているコンテナを停止する。

```
# コンテナ名を指定して停止docker stop mynginx# コンテナIDを指定して停止docker stop 908b44ea2cce
```

### 削除

コンテナを削除する。

```
# コンテナの削除docker rm mynginx# コンテナIDで削除docker rm 908b44ea2cce# コンテナの強制削除docker rm –f mynginx
```

### イメージ

イメージを検索する。

```
# pythonイメージを探すdocker search python# Description省略なしdocker search python –no-trunc
```

イメージをダウンロードする。

```
docker pull python
```

ローカルに保存されているイメージを確認する。

```
docker images
```

コンテナからイメージを作成する。

```
docker commit base flask2
```

イメージにつけたタグを変更する。

```
docker tag flask2 uhddocker/flask2:v1.0
```

イメージをアップロードする。

事前にイメージレジストリにログインしておく必要がある。

```
docker push uhddocker/flask2:v1.0
```

ローカルイメージを削除する。

```
docker rmi uhddocker/flask2:v1.0
```

## Kubernetesコマンド

### Podのライフサイクル

### Podの起動

```
# nginxを作るkubectl run sample –image=nginx
```

### Podの起動中

```
# 作ったPodを確認kubectl get pods# Podの状態が変わる都度出力しながら継続的に確認し続けるkubectl get po –w
```

### Podの削除

```
# 作成したsampleを削除する$ kubectl delete pods sample
```

### Deploymentのライフサイクル

### Deploymentの作成

ここで、deployment・replicaset・podが作成される。

```
kubectl create deployment sample --image=nginx --replicas=3
```

### Deploymentの状態確認

```
kubectl get deployment
```

### Deploymentの削除

```
kubectl delete deployment sampl
```

### Serviceのライフサイクル

### Serviceの作成

```
kubectl expose deployment sample --port=80 --type=NodeType
```

### Serviceの確認

```
kubectl get svc
```

### Serviceの削除

```
kubectl delete svc sample
```
